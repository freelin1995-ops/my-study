## 第 1 章 平面扫描：多边形布尔运算

### 1.1 应用背景：版图操作

在集成电路版图设计中，经常需要对不同掩模层进行布尔组合：
- **AND**：提取两层重叠区域（如一个有源区与多晶硅的交集）
- **OR**：合并多个碎片为连续区域
- **NOT**：从一个层中挖去另一层的区域
- **XOR**：找出两层之间的差异（用于版图对比）

给定数以百万计的多边形，原始 O(n²) 的两两求交算法完全不可行。需要一种能够在大规模输入下高效运行的算法。

### 1.2 问题定义

**输入**：两组多边形（记为 A 和 B），每个多边形以有向边环表示。

**输出**：多边形 A 与 B 在指定布尔运算下的结果多边形集。

**约束**：
- 多边形可能是非凸的、带孔的、含自交的
- 边可以是 45 度/任意角度的（叫曼哈顿/非曼哈顿）
- 规模可达百万条边

### 1.3 算法：平面扫描（Sweep-Line）

KLayout 的核心布尔引擎位于 `dbEdgeProcessor.h/.cc`，使用**平面扫描**（扫描线）算法。

#### 1.3.1 总体流程

```
1. 插入边（带 property 标记所属多边形和层）
2. Phase 1: 交集检测
   - 按 ymin 排序所有边
   - 从下到上逐条带扫描
   - 在每个条带内按 x 递归细分
   - 检测相交边对，记录 CutPoints
3. Phase 2: 边分裂
   - 对有交点的边在交点处切开
   - 保证所有边不再相互贯穿
4. Phase 3: 输出
   - 重新排序已分裂的边
   - 对每个扫描线位置，递交给评估器
   - 评估器根据 wrap count 决定输出
```

#### 1.3.2 扫描过程

假设有一条水平扫描线从 y = -∞ 向 y = +∞ 移动。在任意时刻，扫描线穿过一组边。当扫描线经过一个顶点时（事件点），边的集合发生变化。

KLayout 不是逐点扫描，而是使用**离散 y 条带**：

1. 边按 ymin 排序
2. 扫描窗口为 `[y, y + 1)`（整数坐标下的单位高度）
3. 在当前条带中，按 x 进一步细分：
   - 按 xmin 排序，分组为 x 不交叠的单元
   - 单元内的边两两检测是否相交
   - 相交点记录在 `CutPoints` 结构中

对于曼哈顿几何（所有边轴对齐），使用专门优化的 `get_intersections_per_band_90`，它能利用水平/垂直边的正交性跳过大量不必要的检测。对于任意角度边，使用 `get_intersections_per_band_any`。

#### 1.3.3 边分裂

带交点的边不能直接递交给布尔评估器。`CutPoints` 结构按边记录所有交点：

```
        交点1  交点2
   p1 ───●─────●───── p2
```

分裂后：
```
   p1 ──●   ●──●   ●─ p2
       段1  段2  段3
```

分裂使用 `ProjectionCompare` 沿边方向排序所有交点，然后生成多条无交子边。

#### 1.3.4 布尔逻辑：Wrap Count 模型

布尔评估器 `BooleanOp` 跟踪每条边的 **wrap count**（缠绕计数）。

**基本面**：
- 每条边有一个 property 标记：bit 0 表示层（0 = A, 1 = B），高比特位区分不同多边形
- `m_wcv_n[p]` 记录多边形 p 在扫描线**北侧**的缠绕计数
- 当扫描线穿过一条边时，对应的缠绕计数 +1 或 -1（取决于边方向）

**Inside 判断**：
- `NonZeroInsideFunc`：`inside(wc) = (wc != 0)`，即非零缠绕即为内部

**聚合计数**：
- `wca` = 层 A 所有多边形缠绕计数的总和
- `wcb` = 层 B 所有多边形缠绕计数的总和

**布尔结果**：

| 操作 | 输出条件 |
|------|----------|
| AND | `inside_a(wca) && inside_b(wcb)` |
| OR | `inside_a(wca) \|\| inside_b(wcb)` |
| A NOT B | `inside_a(wca) && !inside_b(wcb)` |
| XOR | 恰好一个为真 |

**输出决策**：
- 结果从 `false → true`：输出 `+1`（边开始）
- 结果从 `true → false`：输出 `-1`（边结束）
- 结果不变：输出 `0`（继续）

#### 1.3.5 水平边处理

水平边（dy = 0）不会跨扫描线，通过 `compare_ns` 机制处理：

1. 如果北侧 inside ≠ 南侧 inside，则需要在水平边上创建输出边
2. `select_edge` 机制允许水平边也被正确纳入布尔结果

#### 1.3.6 性能分析

- **时间复杂度**：O((n + k) log n)，其中 n 是边数，k 是交点数
- **空间复杂度**：O(n + k)
- **曼哈顿优化**：当所有边轴对齐时，交集检测退化为 O(n) 的区间匹配，极大加速

### 1.4 实现总结

| 组件 | 文件 | 类 | 职责 |
|------|------|-----|------|
| 核心引擎 | `dbEdgeProcessor.h/cc` | `EdgeProcessor` | 边管理、扫描调度 |
| 边数据结构 | `dbEdgeProcessor.h` | `WorkEdge`, `CutPoints` | 边的内部表示与交点 |
| 评估器基类 | `dbEdgeProcessor.h` | `EdgeEvaluatorBase` | inside/outside 接口 |
| 布尔操作 | `dbEdgeProcessor.h` | `BooleanOp`, `BooleanOp2` | AND/OR/XOR/NOT |
| 合并操作 | `dbEdgeProcessor.h` | `SimpleMerge`, `MergeOp` | 多边形合并 |
| 边缘选择 | `dbEdgeProcessor.h` | `EdgePolygonOp` | 选多边形内/外部的边 |
| 输出接收 | `dbEdgeProcessor.h` | `EdgeSink`, `EdgeContainer` | 扫描线有序输出 |
| 形状层面 | `dbShapeProcessor.h/cc` | `ShapeProcessor` | 将形状转为边→处理→转回 |

### 1.5 注释及评论

`dbEdgeProcessor` 实现的平面扫描算法与《计算几何》第 2 章的 Bentley-Ottmann 算法同源，但做了 VLSI 领域的特殊适配。主要差异：

1. **离散坐标**：版图使用整数坐标（dbu = databse unit），边可在单位网格上对齐，使得 y 条带天然离散
2. **曼哈顿优化**：EDA 中多数边是轴对齐的，KLayout 为此做了专项优化
3. **Wrap count vs. 奇偶规则**：非零缠绕规则支持嵌套多边形（孔洞）的正确语义
4. **批量处理**：一次扫描可同时运行多个评估器（如同时算 AND 和 NOT）

该实现是 KLayout 几何引擎的基石——上层的 `dbRegion`（多边形集合）、`dbEdges`（边集合）、`dbShapeProcessor` 都最终调用 `EdgeProcessor::process`。

## 第 2 章 约束 Delaunay 三角剖分：网格生成与电阻提取

### 2.1 应用背景：版图寄生提取

在集成电路的物理验证中，需要从版图提取电阻、电容等寄生参数。精确的电阻提取需要将不规则多边形划分为质量可控的三角形网格，再通过有限元方法计算电流分布。另一个应用是多边形网格生成用于仿真。

### 2.2 问题定义

**输入**：一个多边形（可能带孔），以及质量参数（最小角、最大面积等）。

**输出**：多边形内部的三角形剖分，满足：
- 三角形覆盖多边形内部（无间隙、无重叠）
- 原多边形的边作为约束边保留
- 三角形的质量满足给定标准（最小角 ≥ θ，最大面积 ≤ S）

### 2.3 约束 Delaunay 三角剖分（CDT）

KLayout 实现在 `dbPLCTriangulation.h/.cc`，注释明确写道：

> "This class implements a constrained refined Delaunay triangulation using Chew's algorithm."

#### 2.3.1 算法总览

```
triangulate()
  ├── init_box()          创建边界盒（两个三角形）
  ├── make_contours()     插入多边形顶点（增量式）
  ├── constrain()         强制约束边
  └── refine()            Chew 细化
```

#### 2.3.2 初始边界盒

创建一个覆盖多边形包围盒的大三角形（实际是两个三角形组成矩形），保证所有后续插入的点都在当前三角剖分内部，简化了算法：

```
  D ────── C
  |\       |
  | \      |
  |  \     |
  |   \    |
  |    \   |
  |     \  |
  |      \ |
  A────── B
```

#### 2.3.3 增量式点插入

使用 **Bowyer-Watson 范式**的增量式插入：

**点定位**：`find_triangle_for_point` 使用 **greedy walking**（贪心行走）：
1. 从一个启发式种子开始（对已有点进行 √N 采样找到最近的）
2. 从当前三角形中心向目标点方向行走
3. 每次跨越一条边（通过 `find_closest_edge`）

**三种情况**：

1. 点与已有顶点重合 → 返回该顶点
2. 点在已有边上 → `split_triangles_on_edge`：分裂相邻的两个三角形
3. 点在三角形内部 → `split_triangle`：一分为三

```
     v3                    v3
    / \                   /|\
   /   \                 / | \
  /  p  \      →       /  |  \
 /       \            /  p   \
v1───────v2          v1──┴──v2
```

**Delaunay 恢复**：`fix_triangles` 是一个基于队列的边翻转过程。对每条新创建的边，检查其是否满足 Delaunay 条件：

> 边 e 是**非法**的，当且仅当对面顶点在相邻三角形的外接圆内

```
   A───B          A───B
    \ /    →      │   │      （对角线翻转）
     C            │   C

  incircle(C, ABD) ≤ 0  → 合法
  incircle(C, ABD) > 0   → 翻转
```

in-circle 测试（`dbPLCTriangulation.cc:818-839`）使用精确的几何判定。

#### 2.3.4 约束边插入

`constrain()` 将多边形轮廓边作为约束强制插入：

1. **寻找穿越边**：`search_edges_crossing` 沿三角剖分行走进，找到所有与约束边相交的已有边
2. **分裂穿越边**：在交点处将穿越边分裂
3. **边翻转**：通过翻转操作消除非约束穿越
4. **标记约束**：将约束边标记为 `is_segment = true`（不可翻转）
5. **标记外部**：从边界向外 flood-fill，将多边形外部的三角形标记为 `outside = true`
6. **边合并**：`join_edges` 将相邻且共线的约束边合并，减少顶点数

#### 2.3.5 Chew 细化

`refine()` 实现 Chew 的第二算法（Chew, 1993），保证三角网格质量：

**质量参数**（`TriangulationParameters`）：
- `min_b`：半径-最短边比（控制最小角）
- `min_length`：最短边长（防止无限细化）
- `max_area`：最大三角形面积

**步骤**：
1. 遍历所有三角形
2. 对每个不满足质量标准的三角形：
   - 计算其**外接圆圆心**
   - 如果圆心在三角形内部：插入圆心为新顶点
   - 如果圆心在三角形外部：找到最近的边，在其中点分裂
   - 检查并移除落入直径圆内的其他顶点
3. 每次插入后调用 `fix_triangles` 恢复 Delaunay 性质
4. 重复直到所有三角形通过质量标准或达到最大迭代次数

**效果**：
```
  细化前（细长三角形）          细化后（质量均匀）
   ┌─────┐                    ┌──┬───┐
   │   ╱  │                    │ ╱│ ╲ │
   │ ╱    │     →             │╱ │ ╲ │
   ╱      │                    ├──┼───┤
  ╱       │                   │\ │  ╱ │
```

### 2.4 应用：电阻提取

`pexTriangulationRExtractor.h/.cc` 使用 CDT 进行电阻提取：

1. 将电阻图形（多边形）三角化
2. 每个三角形赋予电阻率
3. 计算三角形的传导矩阵
4. 组装全局矩阵并求解，得到总电阻

### 2.5 实现总结

| 组件 | 文件 | 职责 |
|------|------|------|
| PLC 框架 | `dbPLC.h/cc` | Vertex/Edge/Polygon/Graph 网格数据结构 |
| 三角化算法 | `dbPLCTriangulation.h/cc` | Chew 算法的 CDT 实现 |
| 凸分解 | `dbPLCConvexDecomposition.h/cc` | 基于三角化的多边形凸分解 |
| 电阻提取 | `pexTriangulationRExtractor.h/cc` | 基于三角化的电阻提取 |

### 2.6 注释及评论

CDT 是计算几何中最实用的算法之一。KLayout 的实现使用 Chew 细化是业界标准做法（与 Triangle、Shewchuk 算法同源）。greedy walking 点定位虽然最坏情况 O(n)，但实际在均匀网格下期望 O(√n)，且实现简单、无额外数据结构开销。

关键设计选择：
- **浮点坐标**：三角化工作在双精度浮点空间（`db::DPoint`）而非整数空间，避免角度退化
- **参数控制**：通过 `TriangulationParameters` 暴露所有质量参数，灵活适应不同场景
- **保留外部三角形选项**：`remove_outside_triangles = false` 允许先做查询操作再清理

## 第 3 章 四叉树：大规模版图的空间索引

### 3.1 应用背景：高效的空间查询

KLayout 需要频繁执行空间查询：
- "找出所有与给定矩形重叠的图形"
- "找出所有接触某条边的图形"
- 在包含数亿个矩形的版图中

线性扫描不可行，需要空间索引结构。

### 3.2 问题定义

**输入**：一组对象，每个对象有包围盒（box）。

**操作**：
- `insert(obj, box)`：插入对象
- `erase(obj, box)`：移除对象
- `query(box)`：返回所有与查询盒相交/相碰的对象
- `touching(box)`：返回所有与查询盒接触的对象（边界相邻也算）

### 3.3 PR-四叉树

KLayout 的 `dbQuadTree.h/.cc` 实现了一个**点区域四叉树（Point-Region Quadtree）**，与《计算几何》第 14 章的描述一致。

#### 3.3.1 数据结构

- 每个节点代表一个空间区域
- 节点中心 `m_center` 将区域四等分
- 四个子节点（NW, NE, SW, SE）覆盖四个象限
- 每个节点存储一个对象向量 `m_objects`
- 阈值 `thr`（默认 10）：叶子节点满额后分裂

```
        中心点
     ────┼────
     │   │   │
     │ NW│ NE│
     │   │   │
     ────┼────
     │   │   │
     │ SW│ SE│
     │   │   │
     ────┼────
```

#### 3.3.2 插入

```
insert(obj, box):
  n = quad_for(box)        // 判断属于哪个象限
  
  if is_leaf():
    if 还有空间(n < thr):
      m_objects.push_back(obj)
    else:
      split()              // 分裂为内部节点
      重新插入所有对象
  else if n < 0:           // 跨象限 → 存于此节点
    m_objects.push_back(obj)
  else:
    m_q[n]->insert(obj, box)  // 递归到子节点
```

**跨象限处理**：当一个对象的包围盒跨越中心分割线时（`quad_for()` 返回 -1），它不能归入任何一个子节点，此时存储在当前节点。

**根节点扩展**（`grow`）：当插入的对象超出当前根节点的包围盒时，创建一个新的根节点，其包围盒扩大一倍，原根成为其子节点：

```
    旧根     →     新根
   ┌────┐       ┌────┬────┐
   │    │       │    │    │
   │    │       │    │ 旧 │
   └────┘       │    │    │
                └────┴────┘
```

#### 3.3.3 查询

查询使用迭代器模式 `quad_tree_iterator`，带选择器 `Selector`：

- `touching_sel`：选择与查询盒接触或重叠的对象
- `overlapping_sel`：选择与查询盒重叠的对象
- `always_sel`：选择所有对象

遍历是深度优先的，利用 `select_quad` 剪枝——如果一个象限完全不在查询范围内，整个子树跳过。

```
query(box):
  深度优先遍历
  对每个节点：
    对节点的每个对象：
      if selector(query_box, obj.box):  yield obj
    对每个子象限：
      if selector.select_quad(child_box):
        递归进入子节点
```

#### 3.3.4 复杂度分析

| 操作 | 平均 | 最坏 |
|------|------|------|
| 插入 | O(log n) | O(n) |
| 查询（相等大小对象） | O(k + log n) | O(k + n) |
| 擦除 | O(log n) | O(n) |

最坏情况出现在所有对象聚集在一个点附近时，树退化为线性。

### 3.4 与其他空间索引的对比

| 特性 | 四叉树 | kd-树（第 4 章） |
|------|--------|-----------------|
| 划分方式 | 空间中心（规则） | 数据中位数（自适应） |
| 形状 | 正方形区域 | 矩形区域 |
| 插入 | 动态，O(log n) | 不支持动态（需重建） |
| 适合场景 | 均匀分布的对象 | 分布不均匀的对象 |
| 重叠处理 | 跨象限对象存父节点 | 隐式处理 |

### 3.5 注释及评论

四叉树是 KLayout 中最基础的空间索引结构，被 `dbBoxTree`、`dbBoxScanner` 等上层结构依赖。它的优势在于：

1. **均匀空间划分**：与 VLSI 版图的规律性匹配良好
2. **动态插入/删除**：支持编辑操作
3. **根节点自动扩展**：无需预知版图大小
4. **实现简单**：纯模板实现，无外部依赖

缺点是在非均匀分布（如部分区域极密、部分极疏）下性能退化，此时 kd-树更优。

## 第 4 章 kd-树：灵活的多维最近邻查询

### 4.1 应用背景

当需要按任意矩形窗口查询对象、或进行最近邻搜索时，四叉树的规则划分可能不是最优的。`tlKDTree.h` 提供了更灵活的 kd-树。

### 4.2 数据结构

KLayout 的 kd-树是**隐式存储的 kd-树**（Bentley, 1975），不使用传统的节点+指针表示，而是将树编码在一个**扁平 vector** 中。

#### 4.2.1 构造

构造过程 `sort()` / `partial_sort()`：

1. 对当前区间，用 `tl::nth_element` 找到中位数（O(n) 时间选择）
2. 中位数元素成为当前子树的根
3. 计算**右子树的最大值**存入 `m_bounds` 数组（用于剪枝）
4. 重排元素：奇数索引去左子树，偶数索引去右子树
5. 递归构造左右子树，轮换比较维度

```
原始数组（2D，先比 x 再比 y）：
  [(1,5), (3,2), (6,4), (2,7), (5,1), (4,3), (7,6)]

按中位数（x 坐标）重排后：
  [      ,       , (4,3),       ,       ]  根（x 中位数）
   └───── 左子树 (x<4) ─────┘
         └───── 右子树 (x≥4) ─────┘
```

**树布局**（隐式索引）：
- 根在索引 0
- 左子节点在 `index + step`
- 右子节点在 `index + step*2`
- `step` 每层翻倍

#### 4.2.2 范围查询

`kd_tree_it` 迭代器执行深度优先遍历，利用 bound 剪枝：

```
need_visit()：
  检查当前节点的 bound 是否满足搜索条件
  如果不满足，跳过整个右子树

traverse()：
  先 descend 右子树
  再 descend 左子树
  利用 need_visit 跳过不可能包含结果的分支
```

搜索迭代器模式允许客户端使用标准 C++ 迭代器风格：

```cpp
for (auto it = tree.sel_begin(picker, cmp); it != tree.sel_end(); ++it) {
  // it 指向所有匹配 cmp 的对象
}
```

### 4.3 复杂度分析

| 操作 | 时间复杂度 |
|------|-----------|
| 构造 | O(n log n) |
| 范围查询（输出 k 个结果） | O(√n + k)（2D 平均） |
| 插入 | O(n)（需重建） |

### 4.4 与四叉树的对比

kd-树使用**数据驱动的划分**（按中位数分割），因此能更好地适应不均匀分布的数据：

```
四叉树（规则网格）         kd-树（数据自适应）
┌───┬───┐                 ┌───────┬───┐
│  │   │                 │       │   │
│  │   │                 │       │   │
├───┼───┤   →           ├───────┤   │
│  │   │                 │   │   │   │
│  │   │                 │   │   │   │
└───┴───┘                 └───┴───┴───┘
```

四叉树保持均匀格网（查询形状简单），kd-树自适应数据分布（平衡性更好）。

### 4.5 注释及评论

`tlKDTree` 是 KLayout 中唯一用于任意维度空间搜索的通用数据结构（位于 `tl/` 命名空间，不依赖版图类型）。与四叉树相比，它的主要局限是不支持高效的动态插入和删除——插入新对象需要重建整棵树。使用场景主要是静态数据集的批量查询。

## 第 5 章 区间映射：扫描线的状态管理

### 5.1 应用背景

在平面扫描算法中，需要维护一条水平线（或垂直线）上与当前 y 位置相交的所有区间。这些区间上的值（例如"属于 A 层"、"属于 B 层"、"两者都包含"）需要按 x 被高效地查询和更新。

### 5.2 区间映射

`tlIntervalMap.h` 实现了一个从区间 `[i1, i2)` 到值 T 的映射，支持重叠区间的合并语义。

#### 5.2.1 数据结构

区间被存储为一个**排序 vector** `[(i, j, T)]`，按起始坐标 i 排序，区间互不重叠且尽量相邻合并：

```
  存储形式：
  [(0, 10, A), (10, 20, A|B), (20, 30, B)]
```

#### 5.2.2 插入（add）

```
add(i1, i2, value, join_operator):
  1. 二分查找第一个与 [i1, i2) 重叠的区间
  2. 遍历所有重叠区间：
     - 无重叠 → 直接插入新区间
     - 部分重叠（前部）→ 截短前部
     - 部分重叠（后部）→ 截短后部
     - 完全重叠 → 调用 join_operator 合并值
  3. 压缩：相邻且值相同的区间合并
```

示例（插入 `[5, 15)` 且值 = B 到已有映射中）：

```
操作前：[(0, 10, A), (10, 20, C)]
插入：
  [5, 15) ∩ [0, 10) → 分裂 [0,10) 为 [0,5),(A) 和 [5,10),(A join B)
  [5, 15) ∩ [10,20) → 分裂 [10,20) 为 [10,15),(C join B) 和 [15,20),(C)
  压缩：相邻同值可合并
操作后：[(0, 5, A), (5, 15, A|B), (15, 20, C)]
```

#### 5.2.3 查询（mapped）

```
mapped(i):
  二分查找第一个末端 > i 的区间
  如果 i 在该区间内，返回该区间的值
  否则返回默认值
```

### 5.3 应用场景

在 KLayout 中，`tlIntervalMap` 被 `dbEdgeBoolean.h` 使用：
- 表示扫描线上每个 x 区间当前属于哪些多边形
- 快速查询某点是否在多边形内部
- 增量式更新扫描线状态

### 5.4 复杂度分析

| 操作 | 时间复杂度 |
|------|-----------|
| add | O(m)（m 为重叠区间数，最坏 O(n)） |
| mapped | O(log n) |

### 5.5 注释及评论

`tlIntervalMap` 不是一个**区间树**（Interval Tree）。区间树支持 O(log n + k) 的 stabbing query（哪些区间包含某点），而 `interval_map` 更关注**增量的区间合并**——通过 join operator 将重叠部分合并为新值。这相当于维护一个**分段常数函数**（piecewise-constant function），正是扫描线算法所需的数据结构。

## 第 6 章 盒扫描器：矩形交互检测引擎

### 6.1 应用背景

在 DRC（设计规则检查）中，需要找出所有相互接触或重叠的图形对。比如，检查两条金属线之间的间距是否满足最小约束，这需要找出所有距离小于某个阈值的矩形对。

### 6.2 问题定义

**输入**：一组对象，每个对象有包围盒。

**输出**：所有**交互**（interact）的对象对。交互定义为：包围盒在右/上方向扩展 enl 后重叠。

### 6.3 算法：自适应条带扫描

`dbBoxScanner.h` 实现了一种专为 VLSI 版图优化的矩形交互检测算法。

#### 6.3.1 总体流程

```
do_process():
  1. 如果 n ≤ 阈值（默认 100）→ 直接 O(n²) 暴力
  2. 否则：
     a. 按 ymin 排序
     b. y 条带循环：
        - 当前 y = 最早未处理的 ymin
        - 用 partition 移除顶部已超出条带的对象
        - 自适应扩大条带至填充因子达到目标
     c. x 子划分：
        - 按 xmin 排序
        - 自适应按 x 分组
     d. 组内两两测试重叠
     e. 去重（seen set）
```

#### 6.3.2 自适应条带

条带大小不是固定的，而是**动态增长**：

```
条带 = [y, y + Δy)
Δy 由填充因子控制：
  fill_factor = 2.0
  应至少包含 current_size × fill_factor 个新对象后条带才前进
```

这种自适应策略平衡了：
- 条带太小 → 频繁切换，分区开销大
- 条带太大 → 组内对象多，假阳性比较多

#### 6.3.3 重叠检测

`bs_boxes_overlap` 检测两个包围盒是否重叠（带不对称扩展 enl）：

```cpp
bool bs_boxes_overlap(const Box &b1, const Box &b2, Coord enl) {
  return (b1.p1().x() < b2.p2().x() + enl && b2.p1().x() < b1.p2().x() + enl) &&
         (b1.p1().y() < b2.p2().y() + enl && b2.p1().y() < b1.p2().y() + enl);
}
```

`enl = 1` 时检测**接触**（touching），`enl = 0` 时检测**严格重叠**（overlapping）。

#### 6.3.4 双类型扫描

`box_scanner2<Obj1, Prop1, Obj2, Prop2>` 支持两种不同类型的交互检测（如边对多边形），只报告跨类型交互。

### 6.4 应用场景

`box_scanner` 是 DRC 引擎的核心驱动：
- `dbRegionCheckUtils::Edge2EdgeCheckBase` 中的间距检查
- `dbEdgeNeighborhood` 中的邻域查询
- `dbRegionLocalOperations` 中的局部运算

### 6.5 注释及评论

`box_scanner` 可以看作是 `dbEdgeProcessor` 中条带扫描策略的"瘦身版"——它只检测交互而不执行布尔运算。自适应条带加 x 子划分的二阶段策略是经典的 VLSI 扫描算法优化手段，与当年 Magic 布局系统的 corner-stitching 思想一脉相承。

## 第 7 章 多边形工具：偏置、裁剪与形态学操作

### 7.1 应用背景

版图处理中经常需要：
- **Sizing**（缩放）：将多边形放大或缩小一定距离。这是 DRC 中的核心操作——例如生成"有源区"到"栅极"的包围区域
- **点包含测试**：判断某点是否在多边形内
- **轮廓平滑**：去除多边形上的锯齿

### 7.2 偏置操作（Sizing/Minkowski 和）

`dbPolygonTools.h/.cc` 中的多边形偏置基于**Minkowski 和**的近似：

#### 7.2.1 Minkowski 和

多边形 P 与圆盘 B(ε) 的 Minkowski 和定义为：

```
P ⊕ B(ε) = {p + b | p ∈ P, b ∈ B(ε)}
```

对于多边形，这相当于：
- 凸多边形：每条边向外平移 ε，相邻边的交点形成新顶点
- 非凸多边形：做圆角过渡

#### 7.2.2 实现

KLayout 通过**边偏置 + 布尔运算**组合实现 sizing：

```
size(poly, ε):
  1. 每条边向外平移 ε
  2. 边延伸相交生成新多边形
  3. 对尖锐凹角做圆角过渡
  4. 用布尔合并消除自交
```

#### 7.2.3 点包含测试

`inside_poly_test` 实现**射线法**（ray casting）：

```
inside(point, polygon):
  从 point 向右发射水平射线
  统计与多边形边的交点数目
  奇数 → 内部，偶数 → 外部
  处理退化情况（顶点相交、边重叠）
```

### 7.3 裁剪

`dbClip.h/cc` 实现版图区域裁剪，支持：
- 按矩形窗口裁剪多边形
- 按多边形裁剪多边形（借助布尔运算）
- 保留裁剪边界上的层次化信息

### 7.4 形态学操作

- `smooth_contour`：对多边形轮廓做平滑（消除 jogs）
- `extract_rad_from_contour`：从轮廓提取圆角半径
- `cut_polygon`：用直线切分多边形
