# Geometric Pattern Match Using Edge Driven Dissected Rectangles and Vector Space

## 基于边缘驱动的离散矩形与向量空间的几何图样匹配

Jea Woo Park¹`², Robert Todd¹, Xiaoyu Song²
1: Mentor Graphics, 2: 波特兰州立大学

IEEE Transactions on Computer-Aided Design of Integrated Circuits and Systems, 2016

---

**摘要** — 本文提出了一种新颖的图样匹配算法，该算法基于多边形边缘将图样离散为矩形。与其他基于 DRC 的图样匹配算法不同，我们的方案利用简单的 DRC 边缘长度规则来创建用于热点图样描述的矩形。这种方法相比其他方案至少有三个优势：第一，比现有的先进图样匹配工具更快；第二，对图样匹配工程师而言更直观、更简单，易于理解和描述图样；第三，具有良好的并行计算扩展性。我们还展示了如何利用由原点矩形和图样包围盒内其他参考矩形构成的向量空间来改善图样匹配的运行时间。通过采用向量概念，检测不同图样方向时只需迭代一到两次。其他图样匹配技术通常需要迭代 8 次（4 次旋转 × 2 次镜像）来检测所有八种不同方向，我们的方法消除了这些不必要的迭代。

**关键词** — 图样匹配，DFM，可制造性设计，工艺热点，DRC，设计规则检查，光刻

---

### I. 引言

随着芯片设计特征尺寸越来越小、复杂度越来越高，图样匹配不仅在亚波长光刻变化导致的工艺热点检测中至关重要，在双重和三重图形化错误检查中也同样重要，因为它能快速简便地检测错误。

随着图样匹配成为业界热点话题之一，众多研究者致力于提出快速准确的解决方案。主要包括：

**1) 基于机器学习的图样匹配：** 提取热点图样训练人工神经网络，创建监督学习模型用于预测版图中的潜在热点。关键在于创建准确的学习模型以避免误报，但训练时间长，且需要复杂技术来减少误报。事实上，即使误报率相对较低，这种方法也无法完全避免误报。

**2) 基于显式模型的图样匹配：** 创建显式模型来比较目标图样与版图中的图样。例如 Kahng 提出创建对偶图来表示图样，用于过滤掉不匹配的图样。但这种方法同样因固有的建模误差而存在误报问题。相比机器学习方法，它更准确高效。

**3) 基于字符串的图样匹配：** 将字符串匹配技术应用于图样匹配。首先创建网格，每个网格点转换为版图矩阵（与图形重叠为 1，否则为 0），然后将点编码为字符串进行匹配。此方法没有误报问题，但不适合先进工艺设计，因为制造网格尺寸越来越小，计算时间呈指数增长。

**4) 基于 DRC 的图样匹配：** 使用 DRC 规则来识别图样。输入图样被转换为多条 DRC 规则，分析其输出来获得匹配结果。此方法没有误报问题，可用于精确匹配。但随着图样数量增加，会产生大量复杂的 DRC 规则，导致计算成本高昂。我们的方案解决了这些问题。

在这些方案中，基于 DRC 的图样匹配具有明显的优势，并且在工业界被广泛采用，因为它通常比其他方案更快、更准确。

基于这些优势，我们提出了一种新颖的基于 DRC 的图样匹配方案。我们使用简化的边缘驱动 DRC 规则和数学向量概念来实现更快、更准确的结果，并将匹配所有可能图样方向的迭代次数从八次减少到最多两次。

该方案相比现有图样匹配方案至少有三个优势：第一，更快；第二，对图样匹配工程师而言更直观、更简单，易于理解和描述图样，模糊图样特别容易描述且不会增加太多额外开销；第三，具有良好的并行扩展性。DRC 边缘操作以这种特性著称。无论需要匹配的图样数量多少，输入层的所有边缘只需处理一次，因此匹配时间随图样数量增加具有良好的扩展性。

主要贡献总结如下：
- 开发了一种快速图样匹配方法，相比业界领先的商业图样匹配工具及其他方法，实现了显著的加速
- 展示了我们的方法在模糊图样匹配和部分图样匹配中的易用性和灵活性
- 引入了一种实用的向量空间概念来减少图样匹配的迭代次数

### II. 背景

#### A. 设计规则检查

设计规则由工艺工程师提供，以确保设计版图的可制造性。工艺变化和光刻技术的局限性使得每个设计在流片前必须通过 DRC。现代 DRC 规则集很复杂，但始终包含两条最基本的规则：宽度和间距。宽度规则通过定义任何形状的最小宽度来防止狭窄形状的夹断，间距规则通过定义两个形状之间的最小允许距离来防止桥接。这些规则可以用常量或带变量的方程/不等式表示。违反这些规则由 DRC 工具报告违规边缘的位置。

#### B. 边缘操作

DRC 工具的边缘操作用于检查与边缘相关的规则。基本的边缘操作包括 LENGTH、WIDTH、SPACE 和 ANGLE。这些操作可用于生成边缘驱动的离散矩形，如图 2 所示。由于边缘操作是 DRC 工具的核心操作，通常经过速度优化，常采用并行计算。因为我们的方案直接依赖于 DRC 边缘操作，自然也受益于这些优势。

### III. 基于边缘驱动离散矩形的图样匹配

#### A. 图样描述

图样由包围盒内的成员矩形描述。每个成员矩形通过边缘操作推导得出（图 2）。成员矩形可以用以下简单格式描述：

```
<成员名称>
Len 1 == 值 (um 或任意用户单位)
Len 2 == 值
Width == 值 或 Space == 值
```

利用这些信息，我们可以执行 LENGTH、WIDTH 或 SPACE 等 DRC 操作来生成成员矩形。对于可能存在多种分解方式的图样，我们在生成成员矩形时还会额外使用 ANGLE 操作来强制采用唯一的分解方式。

图 3 是一个图样描述示例，包围盒内有六个成员矩形。其中任意一个可以是原点成员，其余任意一个可以是第一个参考成员。

利用原点成员的中心点和第一个参考成员的中心点，我们可以创建向量空间及其他必要信息用于图样匹配。图 4 展示了这一过程。每个成员矩形存储的图样描述信息包括：

1. 向量信息（原点和参考之间的角度和距离）
2. 从原点到参考的平面位置（四个平面或四个轴向之一）
3. 每个成员矩形中心到包围盒的距离（d1, d2, d3, d4）
4. 宽度和高度
5. 布尔值，指示是宽度矩形还是间距矩形

这些信息存储在 PDB（图样描述数据库）中。有了这些信息，我们可以区分任意图样的 8 种不同方向（4 次旋转 × 2 次镜像），消除了检测不同方向同一图样的 8 次不必要迭代（图 5）。

唯一的例外是当向量方向沿轴向或角度为 45 度时，我们需要迭代两次。图 6 解释了沿轴情况下需要迭代两次的原因。

#### B. 图样匹配

利用上述图样描述信息，我们通过 Algorithm 1（暴力搜索算法）在版图上执行图样匹配。如果包围盒内存在非成员多边形，则立即判定为不匹配（图 7）。另一个早期无效化标准是检查包围盒内每个成员的总数是否与 PDB 匹配，不匹配则立即否定（图 8）。

由于 Algorithm 1 使用拓扑扫描线的暴力搜索，尽可能减少 P1（原点成员）的数量对缩短运行时间很有帮助。我们可以通过在创建原点成员时添加与其他参考成员相关的额外边缘操作来实现这一点。

#### C. 分箱搜索网格

分箱搜索网格对于在二维空间中查找交互对象非常高效，通常远快于需要单次扫描所有对象的拓扑扫描线。它使用自适应结构进行矩形搜索。网格先用固定像素大小初始化，当箱内对象数量过多时会重新划分更精细的网格。

网格首先通过 ADD_BIN 方法填充，然后使用 LOCATE_BIN 方法搜索。LOCATE_BIN 的输入是一个搜索矩形，返回与该矩形交互的对象 ID 集合。

由于图样包围盒通常相对于版图范围非常小，在我们的应用中它通常落在一个网格元素内。图 9 说明了这一点。

#### D. 计算复杂度

Algorithm 1 的时间复杂度为 O(n²)，其中 n 是 P1 的数量。Algorithm 2 使用 LOCATE_BIN 替代了扫描线搜索，由于 LOCATE_BIN 的复杂度为 O(k)，其中 k 是搜索框重叠的网格元素数量，而 k ≪ n（通常 k=1），所以 Algorithm 2 在实践中是 O(n)。表 II 显示了两种算法的巨大运行时间差异。

#### E. 模糊图样匹配

我们的方法在模糊图样匹配方面也有优势（图 10）。由于不仅可以使用 == 运算符，还可以使用其他关系运算符（>, ≥, <, ≤）来描述图样，因此可以实现模糊匹配。此时向量空间信息不再有效，我们可以使用包围盒内每个成员的数量来进行模糊图样匹配。

需要注意的是，我们的模糊匹配方法有一个局限性：无法处理从 post-OPC 图样到 pre-OPC 图样的匹配，但基于容忍度的匹配可以轻松实现。

#### F. 部分匹配

我们的方法另一个有趣的用途是部分匹配。通过跳过 Algorithm 1 中的非成员检查步骤（第 16 步），我们可以匹配包含非正交边缘的图样。只要图样中有几个成员矩形，就能进行部分匹配。

其他方法需要定义"不关心区域"来处理部分匹配，而我们的方法无需额外工作，因为部分匹配是自动定义的——如果包围盒内某些多边形或多边形部分未被指定，算法会自动忽略它们。

### IV. 实验结果

实验在 3.7 GHz CPU、32 GB RAM 的 Linux 平台上进行。我们创建了 9 个不同的匹配图样，使用真实的工业版图。

**Algorithm 1 vs Algorithm 2（表 II）：** Algorithm 2 的效率远超暴力搜索的 Algorithm 1。在处理 400 万个 tp9 图样时，Algo 1 需要 205,852 秒，而 Algo 2 仅需 96 秒。匹配成功率为 100%，无假阳性匹配。

**与商业工具对比（表 III）：** 我们的方法与基于顶点哈希的商业 DRC 图样匹配工具相比，快 2\~10 倍。

**与 Yu 等人的方法对比（表 IV & V）：** 在相同的基准测试条件下，我们的方法平均快 20 倍，对于 "Stair1" 图样快 58 倍。作为参考，Su 等人（2015）的方法平均快 5.6 倍，对 "Stair1" 快约 18 倍。

---

### V. 结论

本文提出了一种快速准确的图样匹配新方法。利用超快的 DRC 边缘操作进行边缘驱动离散矩形图样匹配，同时利用数学向量概念避免了检测 8 种不同方向上同一图样的 8 次不必要迭代。我们还展示了模糊匹配和部分匹配等可能的应用。实验结果表明，我们的方法实现了 100% 的准确匹配，且显著快于其他方法。

另一个值得注意的点是，该技术可以轻松扩展到多层图样匹配。例如，如果一个图样包含金属 1、通孔和金属 2 等多个层，我们可以在层内和层间使用边缘操作来推导成员矩形并进行匹配。

### 附录 A — DRC 操作定义

- **LENGTH(E, RO, value)**：返回满足长度约束的边缘。RO 为关系运算符。
- **WIDTH(E1, E2, E3, RO, value)**：测量第一组输入边缘与第二组输入边缘之间朝多边形内部的宽度，返回满足宽度约束的矩形。
- **SPACE(E1, E2, E3, RO, value)**：与 WIDTH 相同，但测量的是朝多边形外部的间距。
- **ANGLE(E, RO, value)**：返回满足角度约束的边缘。
- **OR(P1, ..., Pn)**：返回多边形的并集。
- **NOT(P1, P2)**：返回 P1 中不与 P2 重叠的部分。

### 附录 B — 基于容忍度的匹配

可以通过添加范围关系操作和部分匹配来扩展我们的方法以解决工艺热点的容忍度匹配问题。例如，一个成员可以定义为：
```
Len 1 >= a <= b（Len 1 在长度 a 和 b 之间）
Len 2 >= c <= d（Len 2 在长度 c 和 d 之间）
Width >= e <= f（Width 在宽度 e 和 f 之间）
```
